# Asyre 快速参考卡片

## 一句话说明

基于"雕刻大卫"哲学的迭代式内容创作系统，先快写"石头"，再通过AI推敲删减让内容"显现"。

---

## 核心命令

### 基础使用

```bash
/asyre [主题描述]
```

**示例:**
```bash
/asyre 写一篇关于"为什么年轻人不想上班"的口播稿
```

### 快捷命令

```bash
/asyre quick [主题]      # 跳过确认，直接输出精修版
/asyre critique [内容]   # 仅评价现有内容
/asyre refine [内容]     # 仅对现有内容进行雕刻
```

### 商业化：创作者定制

```bash
/asyre --creator=[创作者ID] [主题]
```

## 六阶段流程

```
┌─────────────────────────────────────────────────────────┐
│ Phase 1: 意图解析 (Intent Parsing)                     │
├─────────────────────────────────────────────────────────┤
│ ✓ 分析主题的意图因子组合                                │
│ ✓ 确定主导目的类                                        │
│ ✓ 推荐钩子类型                                          │
└─────────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 2: 参数推导 (Parameter Derivation)               │
├─────────────────────────────────────────────────────────┤
│ ✓ 匹配推导规则                                          │
│ ✓ 生成关键参数                                          │
│ ✓ 输出执行规则和禁止项                                  │
└─────────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 3: 快写初稿 (Additive Phase) 🪨                  │
├─────────────────────────────────────────────────────────┤
│ ✓ 肆无忌惮，不管质量                                    │
│ ✓ 先把"石头"搬来                                        │
└─────────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 4: 评价文档 (Critique Phase) 🔍                  │
├─────────────────────────────────────────────────────────┤
│ ✓ 标记保留部分（大卫）                                  │
│ ✓ 标记删除部分（石头）                                  │
│ ✓ 提供修改建议                                          │
│ ✓ 评分 + 目标删减率 ≥30%                                │
└─────────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 5: 雕刻修正 (Subtractive Phase) ✂️                │
├─────────────────────────────────────────────────────────┤
│ ✓ 删减30%+                                              │
│ ✓ 输出修正版                                            │
└─────────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 6: 再评价 + 精修 (Refinement Phase) 🏆           │
├─────────────────────────────────────────────────────────┤
│ ✓ 如果评分 < 8/10，继续迭代                             │
│ ✓ 输出待人工审核版                                      │
│ ✓ 收集反馈                                              │
└─────────────────────────────────────────────────────────┘
```

## 13个意图因子

### 认知改变类
- `concept_clarification` - 概念澄清
- `hidden_reveal` - 信息揭示
- `cognitive_disruption` - 认知颠覆
- `knowledge_transfer` - 知识传递

### 态度改变类
- `value_guidance` - 价值观引导
- `persuasion` - 观点说服
- `identity_construction` - 身份认同构建

### 行为改变类
- `action_call` - 行动号召
- `decision_guidance` - 决策引导

### 情感满足类
- `emotional_resonance` - 情感共鸣
- `emotional_catharsis` - 情绪宣泄
- `entertainment` - 娱乐消遣
- `validation` - 认知验证

## 质量标准

### 必须遵守的规则

| 规则 | 说明 |
|:-----|:-----|
| **3秒法则** | 开头3秒内必须传递核心信息 |
| **问题感激发** | 必须让观众感受到"这和我有关" |
| **态度鲜明** | 禁止骑墙，禁止"一方面...另一方面" |
| **口语化** | 句子长度 ≤ 15字，必须能流畅朗读 |
| **删减30%** | 每轮修改必须删掉至少30% |

### 禁止词清单

❌ "大家好，我是..."
❌ "众所周知..."
❌ "相信很多朋友都..."
❌ "今天我们来聊..."
❌ 过长背景铺垫
❌ 模棱两可的表述

## 钩子效力公式

```
效力 = 问题感 × 情绪强度 × 具体性
```

### 8大钩子类型

| 类型 | 公式 | 示例 |
|:-----|:-----|:-----|
| **反直觉** | "很多人以为X，其实Y" | "年终奖发越多，你反越危险" |
| **痛点共鸣** | "你是不是也有过..." | "周日晚焦虑，一想周一就心慌" |
| **冲突悬念** | "X不希望你知道的事" | "这件事可能得罪很多人，但我要说" |
| **数据震撼** | "X%的人都搞错了" | "研究500个失败案例，都犯同错误" |
| **情绪宣泄** | "我真的受够了X" | "受够了'年轻人要吃苦'的话" |
| **身份认同** | "如果你是X类型的人" | "如果你是i人，一定有共鸣" |
| **利益承诺** | "看完就知道如何X" | "学这招，沟通效率翻倍" |
| **故事悬念** | "那天发生了一件事..." | "三年前做个决定，现在回头看..." |

## 常见问题

### Q: 什么时候用 `/asyre`，什么时候直接写？

**A:**
- ✅ 用 `/asyre`: 口播稿、评论、科普、观点文章（需要态度鲜明）
- ❌ 别用 `/asyre`: 纯客观报道、技术文档、学术论文

### Q: 为什么要"快写慢改"而不是一次写好？

**A:**
- 拼凑是自我肯定（舒服但低效）
- 雕刻是自我否定（痛苦但有效）
- "大卫本来就在石头里"

### Q: 评分多少算合格？

**A:**
- 6/10 = 能用，但平庸
- 8/10 = 合格，值得发布
- 10/10 = 极致，但可能过度优化

### Q: 商业化怎么卖？

**A:** 三个层次
1. 咨询：帮创作者定制画像 (高客单价)
2. SaaS：订阅制使用引擎 (规模化)
3. 培训：教授方法论 (长期价值)
