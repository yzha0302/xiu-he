# Asyre

> **Asymmetric Content Creation System**
> 非对称内容创作系统 v2.9

---

## 核心理念

**内容价值 = 人的不可替代性 × AI 的放大效率**

在 AI 时代，人到底该把精力放在哪一层，才能永远不被替代？

**创作哲学**: "雕刻大卫" — 米开朗琪罗说"大卫本来就在石头里"，但前提是那块石头本身就是好石头。初稿就要有灵魂，雕刻只是去冗余。

**框架定位**: 框架用来**启动思考**和**评价优化**，不用来**限制创作**。

---

## 快速开始

### 使用 Claude Skill (推荐)

```bash
/asyre 写一篇关于"为什么年轻人不想上班"的口播稿
```

Skill 会自动执行六阶段流程：
1. **意图解析** - 分析主题的意图因子组合（提供方向感）
2. **参数推导** - 输出关键参数分析（启发创作，不是约束）
3. **自由创作** - 带着方向感自由发挥，**初稿即高质量**
4. **评价文档** - 七层质量公式评分，标记短板
5. **雕刻修正** - 去冗余（10-20%），不是大改
6. **精修审核** - 质量提升轨迹报告，人工反馈

> **重要**:
> - Phase 1-2 的参数分析是**启发**，不是**约束**。给用户看方向，但不框死 AI 创作。
> - Phase 3 不是"快写"，是"自由创作"。这一稿就要有灵魂、有人味、有画面感。
> - 雕刻能删废话，但雕刻不出灵魂。

详见: `.claude/commands/asyre.md` 或 `.agent/workflows/asyre.md`

## 核心创新：意图驱动参数系统

### 传统方式 vs Asyre 方式

```
传统: 形式 (口播稿/长文) → 固定参数组
Asyre: 意图因子组合 → 动态推导参数 → 质量量化评估
```

### 质量公式 (加权模型 v2.3)

**核心逻辑:** 内容价值 = 可信度门槛 × (护城河权重 + 放大器权重)

```
Q = T_gate × Σ(wi × Fi)

T_gate = T > 0.70 ? 1 : 0 (门槛函数)
wi = 第i个因子的融合权重 (理论 + 数据驱动 + 专家判断)
Fi = 第i个因子的得分 (0-1)
```

**三源权重融合:**

```
final_weight = α×theory + β×data + γ×expert

初始比例: α=0.40, β=0.30, γ=0.30
随样本增加，β逐渐提升至0.45，α降至0.25
```

**七层因子 (默认权重):**

| 因子 | 名称 | 层级 | 权重 | 自动化程度 | 护城河强度 |
|:-----|:-----|:-----|:-----|:----------|:----------|
| **T** | 可信度 | 门槛层 | 门槛 | ★★★ | ★★★★★ |
| **E** | 经验+专业 | 护城河 | 31.7% | ○ | ★★★★★ |
| **C** | 深度判断 | 护城河 | 25.7% | ★ | ★★★★★ |
| **F** | 受众匹配 | 放大器 | 14.0% | ★★★ | ★★★ |
| **D** | 表达 | 放大器 | 16.0% | ★★★★ | ★★ |
| **A** | 执行 | 放大器 | 9.3% | ★★★★★ | ★ |
| **B** | 时机 | 放大器 | 3.3% | ★★★★★ | ★ |

**加权逻辑:**
- T层作为门槛过滤器，低于0.70则整体归零
- 其他因子按权重加权求和
- 权重反映因子对内容质量的实际贡献度
- 不同内容类型使用不同权重配置

详见: `Config/quality_weights.json` + `Config/quality_assessment.json`

### 意图因子 (Intent Factors)

每篇内容是多个意图因子的组合：

| 因子 | 描述 |
|:-----|:-----|
| value_guidance | 价值观引导 |
| persuasion | 观点说服 |
| concept_clarification | 概念澄清 |
| hidden_reveal | 信息揭示 |
| emotional_resonance | 情感共鸣 |
| cognitive_disruption | 认知颠覆 |
| action_call | 行动号召 |
| knowledge_transfer | 知识传递 |

### 推导流程

```
意图因子量化 → 规则匹配 → 参数推导 → 执行规则生成
```

详见: `Engine/derivation_process.md`

## 商业化模式

### 核心价值主张

```
Asyre 引擎 (通用) + 创作者画像 (定制) = 个性化内容生产系统
```

### 三层变现路径

1. **咨询服务** — 帮助创作者/企业主识别七层能力边界，定制个人画像
2. **工具 SaaS** — 将意图推导引擎打包为可订阅的创作工具
3. **培训体系** — 教授"意图→参数推导"能力，培养内容策略师

### 创作者画像定制流程

```
Step 1: 分析创作者
  ├─ 意图倾向 (哪些意图因子权重高)
  ├─ 风格调性 (冷峻/温暖/专业/轻松)
  ├─ 禁止词清单 (个人厌恶的表达)
  └─ 受众画像 (目标读者特征)

Step 2: 生成配置文件
  └─ creator_profile.json

Step 3: 使用定制 Skill
  └─ /asyre --creator=[ID] [主题]
```

### 与 Asher_Source_Profile 的集成

- 默认创作者: 使用 `01_Base_Attributes/` 的个人参数
- 其他创作者: 使用独立的 `creator_profile.json`
- 引擎共享: 所有创作者共享同一套推导规则
