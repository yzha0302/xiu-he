# Quests (任务系统)

> **AI 协作者必读**: 在此目录下创建任何内容前，请先完整阅读本 README。

---

## 一、系统定义

Quests 是修荷系统的**外部任务管理模块**，用于记录和追踪所有外部项目、委托、合作事项。

与 `Asher_Source_Profile_v1/` (内部能力) 不同，这里存放的是**具体的外部任务实例**。

---

## 二、任务类别

| 类别 | 目录 | 定义 | 目标 | 示例 |
|------|------|------|------|------|
| **Bounties** | `Bounties/` | 解决问题即获酬 | 即时现金流 | 付费咨询、外包开发、一次性服务 |
| **Side_Quests** | `Side_Quests/` | 战略性任务 | 声誉、人脉、经验 | 帮助合作伙伴、作品集项目、战略性无偿工作 |
| **Power_Leveling** | `Power_Leveling/` | 纯支持/合作 | 利他、帮助朋友 | 技术支持、建议咨询、无期望回报的帮助 |

## 三、任务创建协议 (AI 必读)

### 3.1 创建流程

当用户要求创建新任务时，AI 需要执行以下步骤：

```
Step 1: 确定任务类别
        ↓
Step 2: 生成任务代码 (Quest Code)
        ↓
Step 3: 创建任务文件夹
        ↓
Step 4: 创建任务文档 (QUEST.md)
        ↓
Step 5: 返回任务信息给用户
```

### 3.2 任务代码生成规则

**格式:** `{类别前缀}{日期}{序号}`

| 类别 | 前缀 |
|------|------|
| Bounties | `BNT` |
| Side_Quests | `SQT` |
| Power_Leveling | `PWL` |

**日期格式:** `YYMMDD`

**序号:** 当日第几个任务，两位数字 `01-99`

**示例:**
- `BNT26012601` — 2026年1月26日创建的第1个赏金任务
- `SQT26012501` — 2026年1月25日创建的第1个支线任务
- `PWL26012301` — 2026年1月23日创建的第1个代练任务

### 3.3 文件夹命名规则

**格式:** `{日期}_{委托方}_{简述}_{任务代码}`

- **日期:** `YYYY-MM-DD`
- **委托方:** 人名/公司名/组织名（英文或拼音，无空格）
- **简述:** 2-5个词描述任务内容（英文或拼音，用下划线连接）
- **任务代码:** 按上述规则生成

**示例:**
```
Bounties/
├── 2026-01-26_Wanda_AI_Consulting_BNT26012601/
├── 2026-01-25_Zhangsan_Contract_Review_BNT26012501/
└── ...

Side_Quests/
├── 2026-01-24_Feisheng_Book_Collaboration_SQT26012401/
└── ...

Power_Leveling/
├── 2026-01-23_Lisi_Tech_Support_PWL26012301/
└── ...
```

### 3.4 任务文档模板 (QUEST.md)

每个任务文件夹内必须包含一个 `QUEST.md` 文件：

```markdown
# {任务标题}

> **Quest Code:** {任务代码}
> **Category:** {Bounty | Side Quest | Power Leveling}
> **Status:** {Active | Completed | On Hold | Cancelled}
> **Created:** {YYYY-MM-DD}
> **Updated:** {YYYY-MM-DD}
```

## 四、AI 创建任务示例

**用户输入:** "帮我创建一个任务，万达找我做 AI 咨询"

**AI 执行:**

1. **确定类别:** 付费咨询 → `Bounties`

2. **生成任务代码:**
   - 今天: 2026-01-26
   - 检查当日已有任务数 → 假设是第1个
   - 代码: `BNT26012601`

3. **创建文件夹:**
   ```
   Bounties/2026-01-26_Wanda_AI_Consulting_BNT26012601/
   ```

4. **创建 QUEST.md:** 按模板填写已知信息

5. **返回确认:**
   ```
   已创建任务:
   - 代码: BNT26012601
   - 路径: Quests/Bounties/2026-01-26_Wanda_AI_Consulting_BNT26012601/
   - 文档: QUEST.md (已创建模板，请补充详细信息)
   ```

## 五、状态管理

| 状态 | 含义 |
|------|------|
| `Active` | 进行中 |
| `Completed` | 已完成 |
| `On Hold` | 暂停中 |
| `Cancelled` | 已取消 |

---

## 六、与修荷系统的整合

- **调用能力:** 执行任务时可引用 `Asher_Source_Profile_v1/02_Skill_Tree/` 的技能模块
- **产出存档:**
  - 通用内容产出 → `Content/`
  - 任务专属产出 → 任务文件夹内
- **知识沉淀:** 任务复盘中的经验教训可提炼存入 `Asher_Source_Profile_v1/`

---

## 七、注意事项

1. **命名一致性:** 文件夹名和任务代码一旦创建不要修改
2. **日志习惯:** 每次工作后更新执行日志
3. **完成复盘:** 任务结束后务必填写复盘部分
4. **敏感信息:** 涉及金额、合同等敏感信息谨慎记录
