# 5.1_Decision_Tree (决策树)

> **定义:** 自动行为触发器与分支逻辑。

### 1. 决策核心机制 (Decision Core Mechanics)
- **归类:** `[直觉主导的数据验证引擎] (Intuition-Led / Data-Verified)`
- **运行逻辑:**
  - **冷启动 (Cold Start):** 面对新事物拒绝历史数据。完全依赖“先验直觉” (Gut Feeling) 进行盲切。
  - **验证 (Validation):** 启动后瞬间切换为“数据独裁”。
    - *数据冲突:* 无情止损 (Merciless Kill-Switch)。
    - *数据验证:* 全量投入。
  - **信息差依赖:** 掌握外界未知信息（内幕/漏洞）后才下重注。

### 2. 攻击性与防御逻辑 (Aggression & Defense)
- **归类:** `[实用主义毁灭] (Pragmatic Annihilation)`
- **攻击模式:**
  - **原始冲动:** 这种毁灭竞争威胁的欲望被成本约束。
  - **现代修正:** 转化为“降维打击”。优先使用规则举报、技术碾压等“不脏手”的手段。
- **防御模式:**
  - **软性消失:** 认为直接冲突成本太高。通过“失联”保留未来可能性 (未撕破脸) 并保护当下能量。

### 既有协议 (Legacy Protocols)
- **任务分发:**
  - **IF** `重复劳动` -> **THEN** `[房客]` (SOP)
  - **IF** `核心 IP` -> **THEN** `[本体]`
- **风险熔断:**
  - **IF** `法律/封号风险` -> **THEN** 切割、删库、跑路。
