# 00_System_Manifesto (系统宣言) v2.2

> **本系统是什么:** Asher 的数字分身与思维外骨骼——一个能够在我状态不佳时"教导我自己"的第二大脑。
>
> **Version:** v2.2 | Updated: 2026-01-24

## 一、核心功能

### 这个系统解决什么问题？

**问题:** 人的状态是波动的。疲惫、情绪低落、信息过载时，决策质量会大幅下降。但"状态最好的我"曾经想清楚过很多事情——这些思考如果不记录下来，下次遇到类似情况又要从头想。

**解决方案:** 把"状态最好时的思考"外化、结构化存储，让"状态不好时的我"可以随时调用。

### 两个核心角色

| 角色 | 功能 | 使用场景 |
|------|------|----------|
| **内在顾问** | 当我困惑时，告诉我"状态最好的我会怎么做" | 决策、情绪低落、方向迷失 |
| **数字分身** | 代替我执行已经想清楚的事情 | 写作、分析、重复性认知任务 |

### 核心对话模式

```
我: "如果是状态最好的我，现在会怎么做？"
系统: [调用相关模块] → 给出建议
```

### 各目录用途

| 目录 | 存什么 | 什么时候用 |
|------|--------|------------|
| `01_Base_Attributes/` | 性格、认知模式、优缺点 | 需要了解"我是什么样的人"时 |

| `05_Protocols/` | 决策流程、执行标准 | 需要知道"遇到X怎么办"时 |
| `06_Consciousness_Stream/` | 原始想法、灵感记录 | 需要追溯思考脉络时 |

## 三、技能模块系统

### 工作原理

1. **识别能力** — 我会做某件事（如"写论文"、"谈判"）
2. **拆解流程** — 把模糊的能力变成具体的步骤
3. **存入模块** — 写成标准格式，存入 `02_Skill_Tree/Modules/`
4. **需要时调用** — 告诉系统"我要做X"，系统加载对应模块辅助执行

### 模块标准格式

```markdown
# [技能名称]
> **触发条件:** 什么时候用
> **输入:** 需要提供什么
> **输出:** 会得到什么

## 逻辑
（思考过程）

## 步骤
（具体操作）
```

### 当前主要模块

| 类别 | 模块示例 | 用途 |
|------|----------|------|
| 商业 | 谈判、估值、竞品分析 | 商业决策 |
| 合同 | 合同攻防、风险识别 | 法律相关 |
| 内容创作 | 脚本写作、Hook技术 | 内容生产 |
| 哲学 | 蜕升世界观 | 价值观参考 |

## 四、Workflows 工作流

> **位置:** `.agent/workflows/`
> **导引:** `.agent/WORKFLOWS_GUIDE.md`

### 什么是 Workflow？

Workflow = 把重复性的认知任务封装成一键可调用的标准流程。

### 当前可用 Workflows

| 类别 | 命令 | 功能 |
|------|------|------|
| **知识管理** | `/learn` | 学习新内容 → 生成标准文档 |
| | `/fact-check` | 核查并补充知识 |
| | `/synthesize` | 整合多个文档 |
| | `/sync-graph` | 同步到知识图谱 |
| **会议处理** | `/raw-transcript` | 保存原始对话 |
| | `/meeting-analysis` | 深度分析会议 |
| | `/distill-discussion` | 提炼讨论要点 |
| **内容创作** | `/create-script` | 创作脚本 |
| | `/expand-book` | 扩写书稿 |
| **工具** | `/export-pdf` | 导出PDF |
| | `/render-site` | 生成网站 |
| | `/analyze-contract` | 分析合同 |

### Workflows 会持续扩展

- 每发现重复的认知流程 → 封装为新 Workflow
- 每个 Workflow 会根据使用反馈持续优化
- 目标：90% 的日常认知任务可通过调用 Workflow 完成

## 五、使用原则

| 原则 | 说明 |
|------|------|
| **不要猜测** | 不确定的事情要问，不要脑补 |
| **高保真** | 输出要符合 Asher 的风格和价值观 |
| **解决问题** | 一切为了解决实际问题，不为整理而整理 |
| **先加载再执行** | 执行任务前先加载相关模块 |

## 六、哲学背景（参考）

本系统的设计基于一套完整的世界观，完整内容见：
- **`02_Skill_Tree/Modules/Philosophy/Ascension_Worldview/Ascension_Worldview.md`** — 《蜕升》

核心理念摘要：
- **蜕升** = 通过外挂系统扩展认知能力，而不是试图记住一切
- **附能** = 我不需要成为专家，只需要能调用专家级的能力
- **器与道** = AI 接管"怎么做"，人类守护"为什么做"
