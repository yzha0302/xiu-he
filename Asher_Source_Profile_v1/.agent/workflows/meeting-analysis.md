# /meeting-analysis - 会议深度分析工作流

> [!NOTE]
> **目的:** 将原始对话转化为结构化的知识输出，提取观点、达成共识、制定行动方案

## 触发条件
- 手动触发: `/meeting-analysis [Raw_Transcript文件路径]`
- 重要会议后需要深度复盘
- 需要提取可执行的行动方案
- 需要对外分享或引用

---

## 工作流程

### Step 1: 读取原始对话
// turbo
```
输入: 00_Archive/Raw_Transcripts/ 下的原始对话MD文档
要求: 完整阅读，理解对话流程和核心议题
```

### Step 2: 元认知分析
// turbo
回答以下问题：
1. **讨论背景**: 这次对话发生的context是什么？
2. **核心问题**: 这次对话试图解决什么问题？
3. **讨论脉络**: 对话是如何从A话题流转到B话题的？
4. **关键转折**: 哪些moment产生了breakthrough？

输出:
```markdown
## 1. 元认知 (Metacognition)

### 讨论背景
[3-5段描述]

### 核心问题
[Mermaid流程图展示核心问题的分解]

### 讨论脉络
[Mermaid流程图展示话题流转]
```

### Step 3: 观点还原
// turbo
**识别模式:**
- `**说话人:** [内容]` → 提取核心观点
- 重复出现的概念 → 识别为核心主题
- 带有"我觉得"、"本质上"、"重要的是"等标志词 → 识别为观点

**对每个核心观点进行WWHIS分析:**
```markdown
### X.X 观点标题

**内容 (What):**
> [引用原文核心句]

**说话人:** [谁说的]

**形成原因 (Why):**
[为什么会形成这个观点]

**边界与前提 (Context):**
- **适用场景:**
- **不适用场景:**
- **隐含假设:**

**衍生思考 (Implication):**
[这个观点会导致什么]

**实现路径 (How):** [如适用]
[具体如何执行]

**逻辑链可视化:**
[Mermaid图展示逻辑关系]

**举例 (原文保留):**
[从原文中提取的例子]
```

> [!IMPORTANT]
> 观点数量建议: 10-15个核心观点。过多会失焦，过少会遗漏重要内容。

### Step 4: 事实核查
// turbo
**核查内容:**
1. 引用的书籍、理论、研究
2. 提到的数据、统计
3. 历史事件
4. 专业术语的使用

**输出格式:**
```markdown
## 3. 事实核查 (Fact-Checking)

### 引用核验汇总

| 观点 | 引用内容 | 核查结果 | 备注 |
|------|----------|----------|------|
| X.X | 《书名》 | ✅ 准确 | 作者、出版年份等 |
| X.X | 某个数据 | ⚠️ 部分准确 | 实际情况是... |
| X.X | 某个说法 | ❌ 不准确 | 更正为... |

### 概念准确性检查
[检查专业术语的使用是否准确]
```

### Step 5: 维度梳理
// turbo
**识别讨论维度:**
常见维度包括：
- 认知维度（如何理解问题）
- 创作维度（如何产出内容）
- 传播维度（如何触达受众）
- 商业维度（如何变现）
- 哲学维度（本质思考）
- 执行维度（具体行动）

**每个维度包含:**
```markdown
### X.X 维度名称

[Mermaid流程图展示该维度的核心逻辑]

**核心洞察:**
[从这个维度得出的关键洞察]
```

### Step 6: 共识提取
// turbo
**识别共识类型:**
1. **强共识**: 双方明确同意且达成一致
2. **待确认问题**: 需要进一步讨论或验证
3. **分歧点**: 双方观点不一致

**输出格式:**
```markdown
## 共识清单 (Consensus Summary)

### 强共识 (明确达成一致)

| # | 共识内容 | 支撑依据 | 行动方向 |
|---|----------|----------|----------|
| 1 | ... | 观点X.X | ... |

### 待确认问题

| 问题 | 重要性 | 建议 |
|------|--------|------|
| ... | 高/中/低 | ... |

### 分歧点

| 分歧内容 | 观点A | 观点B | 共识/建议 |
|----------|-------|-------|-----------|
| ... | ... | ... | ... |
```

### Step 7: 行动方案制定
// turbo
**时间维度规划:**
- 短期行动（1-3个月）
- 中期行动（3-6个月）
- 长期目标（6-12个月）

**输出格式:**
```markdown
## 5. 最终共识与行动方案

### 5.1 核心共识
[列表形式]

### 5.2 行动方案

#### 短期行动（1-3个月）
| 行动项 | 负责人 | 优先级 | 具体步骤 |
|--------|--------|--------|----------|
| ... | ... | P0/P1/P2 | ... |

#### 中期行动（3-6个月）
...

#### 长期目标（6-12个月）
...

### 5.3 关键决策点
[需要做出选择的关键点]

### 5.4 风险与应对
[识别可能的风险和应对措施]
```

### Step 8: 系统关联
// turbo
**建立知识链接:**
```markdown
## 6. 系统关联 (System Links)

### 输入依赖
**前置讨论/知识:**
- [链接到相关文档]

### 输出影响
**将影响的后续模块:**
- [待创建/待更新的文档]

[Mermaid图展示知识流动]
```

### Step 9: 术语表生成
// turbo
**提取新术语:**
从对话中识别出：
- 新创造的概念
- 重要的专业术语
- 需要明确定义的词汇

```markdown
## 7. 术语表 (Glossary)

| 术语 | 定义 | 首次出现 |
|------|------|----------|
| ... | ... | 观点X.X |
```

### Step 10: 元数据记录
// turbo
```markdown
## 8. 元数据 (Metadata)

### 讨论统计
- **原始字数**: XX 字
- **处理后字数**: XX 字
- **核心观点数**: XX 个
- **引用核验数**: XX 个

### 讨论特点
[标注这次讨论的特色]

### 后续跟进
[需要follow-up的项目]
```

## 文档结构标准

**完整结构 (10个部分):**
1. 元信息 + 标题
2. 共识清单
3. 元认知
4. 观点还原 (核心部分)
5. 事实核查
6. 维度梳理
7. 最终共识与行动方案
8. 系统关联
9. 术语表
10. 元数据

## 质量标准

| 检查项 | 要求 |
|--------|------|
| 观点提取 | 10-15个核心观点 |
| 每个观点包含 | What/Who/Why/Context/Implication/How |
| Mermaid图数量 | 至少5个 |
| 事实核查 | 所有引用都需核查 |
| 共识明确度 | 清晰区分强共识/待确认/分歧 |

## 常见问题

### Q1: 如果对话很长（3小时+），如何处理？
**A:** 建议分段分析，或聚焦于核心议题，其他部分简化处理

### Q2: 如何判断哪些是"核心观点"？
**A:**
- 重复出现3次以上的概念
- 引发对方长篇回应的观点
- 明确带有"重要的是"、"本质上"等标志词
- 最终形成共识或行动的观点

### Q3: 如果双方观点矛盾怎么办？
**A:** 在"分歧点"部分如实记录，不强行调和

### Q4: Mermaid图不会画怎么办？
**A:** 使用AI辅助生成，或使用简单的流程图结构
