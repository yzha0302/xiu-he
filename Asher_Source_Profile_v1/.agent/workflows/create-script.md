# /create-script 视频脚本创作工作流

---
description: 创建深度科普/评论类视频脚本，具备反直觉钩子、跨领域引用、通俗易懂的特点
---

## 核心原则

> **好的作品需要结构，但好的创作者的结构不能是死的。**

本工作流提供**思考框架**而非**固定模板**。每篇内容都应该根据选题特性找到最适合的表达方式。

---

## 第一步：选题发散

**输入:** 用户提供的热点/方向/关键词

**AI思考方向:**
- 大众对此的常识/刻板印象是什么？
- 有什么反例/例外/隐藏真相？
- 谁是真正的受益者/受害者？
- 历史上有没有类似的事？
- 其他领域有没有可类比的现象？

**输出:** 多个可能的切入角度，**不限于5个**，根据选题灵活发散

---

## 第二步：确定核心洞察

**用户选择或指定角度后:**

**AI需要明确:**
1. **核心洞察是什么？** (一句话能说清楚)
2. **观众看完应该获得什么？** (认知升级/情绪/行动)
3. **什么样的结构最适合表达这个洞察？**

**注意:** 结构服务于内容，不是内容服务于结构。可能的结构包括但不限于:
- 时间线叙事
- 多方博弈分析
- 层层剥洋葱
- 对比分析
- 问题-解答
- 故事驱动
- 论点-论据-升华
- **或者完全不同的结构**

---

## 第三步：素材与知识检索

**根据选题需要，收集:**
- 必要的背景知识
- 支撑论点的数据/案例
- 增强说服力的引用
- 可能的跨领域类比

**知识来源:**
- 现有知识库模块
- 网络搜索
- 用户提供的素材

---

## 第四步：创作

**风格原则 (非死规范):**

| 原则 | 说明 |
|------|------|
| **短句优先** | 能短则短，但不刻意 |
| **人话表达** | 像对朋友讲故事 |
| **有画面感** | 让人能"看见" |
| **节奏感** | 有张有弛 |
| **信息可靠** | 能查证的要查证 |

**创作时思考:**
- 开头能不能更抓人？
- 中间会不会让人走神？
- 结尾有没有留下印象？

---

## 第五步：自检与迭代

**自问:**
- 核心洞察表达清楚了吗？
- 有没有多余的废话？
- 有没有遗漏的关键信息？
- 读起来顺畅吗？
- 普通人能看懂吗？

**根据检查结果自行修改，然后提交给用户**

---

## 第六步：学习反馈

**当用户修改或评价后:**
1. 分析用户的修改意图
2. 记录"什么是好的/什么是不好的"
3. 更新 `_Feedback_Log.json`
4. 下次创作时参考

## 使用方式

```
/create-script [选题关键词]
```

AI会根据选题特性灵活执行上述步骤，**不生搬硬套任何固定模板**。
